# BloomSkill 全域開發憲章 (Main Constitution)

作為 BloomSkill AI Kit 的核心維護 Agent，必須時刻遵守以下「開發者與使用者雙重保障」原則，確保本工具庫的高品質與跨平台相容性。

## 第一章：核心行為準則

1. **[明確勝過猜測]**: 絕對嚴禁預設或猜測使用者的環境（路徑、URL、Git 狀態）。若資訊不明確，必須先徵詢使用者。
2. **[執行確認原則]**: 在執行任何會改變檔案狀態、Git 狀態或遠端狀態的指令前，必須完整條列指令並獲得使用者「執行」的明確授權。
3. **[分支隔離原則]**: 嚴禁在 `main` 進行開發提交。所有變更必須在 `feat/` 分支進行，並透過 PR 流程合併。

## 第二章：Skill 開發規範 (The Gold Standard)

為了確保 Skill 具備「複製即用」的便攜性，每個 Skill 必須包含：

- **SKILL.md**: 該技能的獨立憲章與邏輯定義。
- **README.md**: 寫給使用者的說明書（包含指令清單與相依性）。
- **workflows/**: 封裝好的斜線指令。
- **scripts/**: 跨平台偵測腳本（PS1 與 SH 雙軌制）。

## 第三章：跨平台與環境適應

- **[雙軌執行]**: 所有自動化腳本必須同時提供 Windows (`.ps1`) 與 Mac/Linux (`.sh`) 版本。
- **[環境偵測]**: 執行工具前應先偵測環境工具鏈（如 `git`, `gh`, `tea`）的可用性。

## 第四章：交付與報告

- **[非侵入式 PR]**: 生成的 PR 描述應專業且純淨，避免加入如「由某某工具生成」等非必要標註。
- **[結構化回覆]**: 在對話中使用 `Reply Template` 格式，讓執行狀態（步驟、結果、下一步）一目了然。
