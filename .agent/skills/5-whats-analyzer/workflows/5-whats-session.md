# 啟動問題分析 (/5-whats-analyzer)

1. **[鎖定目標]**: 詢問使用者目前的困擾或想要分析的事件。
2. **[對話循環]**: 
   - 依序提問 **Trigger**, **Gap**, **Boundary**, **Pattern**, **Pivot**。
   - 確保每次提問前摘要前一階段的洞察。
3. **[報告生成]**: 
   - 收集完 5 個層次的回答後，自動彙整為 Markdown 格式的正式報告。
   - 將報告存檔至：`output/Consultation_Report_[YYYYMMDD].md`。
4. **[收尾]**: 輸出諮詢師總結並詢問是否需要為此建立 GitHub Issue 記錄後續行動。
