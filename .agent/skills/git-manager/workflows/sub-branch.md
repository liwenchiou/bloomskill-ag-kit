# 建立分支 (/sub-branch)

此工作流用於在開始開發前，建立符合 Git Manager 憲章規範的分支。

## 執行邏輯

1. **檢查當前分支**: 確認當前是否在 `main` 或穩定分支，並確保工作目錄是乾淨的。
2. **詢問分支資訊**:
   - 詢問開發類型 (如：`feature`, `bugfix`, `hotfix`, `docs`, `refactor`)。
   - 詢問需求單號 (如無單號則略過該區段)。
   - 詢問簡短的英文描述。
3. **組合分支名稱**:
   - 格式為：`類型/單號-描述` (例如：`feature/123-login-api`)。
   - 若無單號格式為：`類型/描述` (例如：`bugfix/fix-typo`)。
4. **⚠️ [建立確認]**: 列出準備執行的指令，例如：`git checkout -b feature/123-login-api`。
5. **許可執行**: 待使用者回覆「確認」後，建立並切換至新分支。
