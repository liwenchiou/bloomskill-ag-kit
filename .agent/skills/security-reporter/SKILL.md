# 🎯 核心目標

結合 `/security-report` 與 `/fact-check` 兩大工作流，確保每月寄出給全公司同仁的資安通報不僅具有時效性，其內容所述的詐騙案例與手法均經過權威機構的二次查核與證實，保證資訊 100% 正確不誤導。

# 🛠️ 執行指令：執行「核實版資安通報」、「產生資安通報」或「/verified-security-report」

當使用者要求透過此 Skill 產出通報時，AI 必須嚴密執行以下三個階段：

## 階段一：草稿生成 (執行本 Skill 內部的 workflows\security-report.md)

1. 啟動情報蒐集，鎖定本月最新的重大資安新聞或網路詐騙手法。
2. 依照 `templates\資安通報範例.md` 的結構，產出本月份的 Markdown 資安通報草稿。
3. 產生草稿檔案至 Skill 目錄下的：`output\Security_Bulletin_[YYYYMM].md`。
4. **暫停並提示**：「✅ 第一階段草稿已產生。即將進入第二階段：對草稿內的【所有詐騙案例】進行深度事實查核，請問是否繼續？」

## 階段二：嚴格查核 (執行本 Skill 內部的 workflows\fact-check.md)

1. 等待使用者確認後，萃取草稿中所有提到的「詐騙手法」與「真實案例」。
2. 將這些案例作為輸入，自動在背景觸發內部的查核機制（利用 Web Search 搜尋 `165防騙網`、`台灣事實查核中心` 等機構）。
3. **查核結果判定**：
   - 若草稿內的案例均為 **✅ 真實訊息 (True)**：直接進入階段三。
   - 若發現有 **⚠️ 部分錯誤** 或 **❌ 不實訊息**：
     AI 必須主動介入修改草稿，剔除假消息，換上真正被官方證實的新案例。

## 階段三：最終發佈與報告

1. 確認草稿已經過 100% 查驗與必要修正後，輸出最終版的 `output\Security_Bulletin_[YYYYMM].md`。
2. 產出報告：
   - 列出本次查核過的案例來源連結（供使用者備查）。
   - 提示使用者：「🎉 本月【雙重核實版】資安通報已竣工！檔案已存放在 Skill 目錄下的 output 資料夾。」
