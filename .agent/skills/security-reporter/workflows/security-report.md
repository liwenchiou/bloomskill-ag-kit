# 🎯 核心目標

協助使用者產出專業、具時效性的公司內部資安通報。通報內容必須參考 Skill 目錄下 `templates\資安通報範例.md` 的結構，並整合當月最新的詐騙案例。

# 🛠️ 執行指令：/security-report

1. **情報蒐集階段**：
   - 執行 Web Search，關鍵字包含：「台灣 最新 詐騙 手法 165」、「BEC 商業電子郵件詐騙 案例 2026」、「資安通報 2026」。
   - 鎖定本月份傳出的重大資安事件或常見社交工程手法。

2. **報告產出階段 (初稿生成)**：
   - 使用 Skill 目錄下 `templates\資安通報範例.md` 作為結構模板。
   - **更新內容**：
     - **通報日期**：填寫當前年月。
     - **背景說明**：撰寫 2-3 段最新的詐騙趨勢摘要。
     - **常見詐騙手法**：列出至少兩項本月最具代表性的手法細節。
     - **防護原則**：保留「三不一要」等核心原則，並視案例增加針對性建議。

3. **事實查核驗證階段 (強制執行工作流 `workflows\fact-check.md`)**：
   - 從生成的初稿中，萃取出所有的「詐騙手法」與「案例」。
   - **必須自動觸發 `/fact-check` 邏輯**：利用 Web Search 去比對「台灣事實查核中心」或「165 全民防騙網」的最新通報。
   - **驗證與修正規範**：
     - 若查出該案例為「證據不足」或已被證實是「多年前的假訊息 / 誇大謠言」：AI 必須主動刪除該案例，並換上另一個真正被核實的新手法。
     - 確認所有案例均為「✅ 真實訊息」。

4. **最終存檔與發佈回饋**：
   - 將經過 100% 驗證的最終報表，存檔至 Skill 目錄下：`output\Security_Bulletin_[YYYYMM].md`。
   - 向使用者**主動回報驗證結果**，例如：「已完成事實查核，列出的案例 X 與 Y 均已在 165 防騙網證實為本月真實驗案，參考連結為 [來源]。」
   - 提示使用者：「🎉 雙重核實版資安通報已產出，您可以安心將內容發送 Mail 給同仁。」

# 📅 週期性提醒

- 在每月第一週執行 `/daily-report` 時，AI 應主動檢查該月是否已在 `output\` 產出 `Security_Bulletin_[YYYYMM].md`。
- 若無，則在日報結尾提示：「💡 新月份已開始，建議執行 `/security-report` 產出本月資安通報。」
