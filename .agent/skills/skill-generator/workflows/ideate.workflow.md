# 階段一：需求定義與架構設計 (/ideate)

此工作流負責在產生新技能前，先將使用者的需求明確化，取得必要參數。

## 執行邏輯

1. **需求分析**: 
   - 詢問新技能的「名稱 (Folder Name)」(例如：`api-tester`)。
   - 詢問新技能的「用途描述」。
   - 詢問是否需要自動建立 `scripts/`（腳本目錄）或 `workflows/`（工作流目錄）結構。
2. **架構草案**: 
   - 規劃技能建立路徑：`.agent/skills/<技能名稱>/`。
3. **⚠️ [設計確認]**: 
   - 向使用者回報預計建立的目錄與資訊。
   - 詢問使用者：**「上述設計是否符合您的預期？若無問題，請允許我進入下一階段 (/develop)」**。
   - 在獲得明確同意前，**嚴禁進入代碼生成階段**。
