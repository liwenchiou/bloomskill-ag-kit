# skill-tester 核心憲章 (Constitution)

> [!CAUTION]
> ### 👑 至高無上原則 (Supreme Directive)
> **只要觸發與執行本技能，本文件的每一項條款即為最高判斷標準。Agent 的任何決策與操作，無論使用者的指令為何，都絕對不得凌駕或違背本憲章之規範。**

> [!IMPORTANT]
>
> ### ⚡ 核心開發規範 (Core Principles)
>
> **1. PDCA 執行生命週期 (Plan-Do-Check-Act)：**
>
> - **[Plan / /ideate]**: 詢問目標 Skill，讀取並解析其 `SKILL.md` 與 `workflows/`，整理接收標準 (AC)。
> - **[Do / /develop]**: 依憲章守則與工作流自動生成對應測試案例清單。
> - **[Check / /validate]**: 審查測試案例是否涵蓋全部 AC 項目，無遺漏。
> - **[Act / /test]**: 於沙盒環境執行，產出報告。若失敗退回 Do 修正，自癒上限 **3 次**。超限輸出 `[ERROR_LOG]`，觸發 **[RULE-005]**。
>
> **2. 沙盒隔離原則：**
>
> - 所有測試行為必須限制在 `/tmp/skill-sandbox/` 隔離目錄內執行。
> - **嚴禁**直接讀寫或修改 `.agent/skills/` 下的任何原始技能檔案。
> - 測試完畢後，必須自動執行 `scripts/sandbox-cleanup.sh` 清除殘留。
>
> **3. 獨立封裝與獨立運作 (Standalone)：**
>
> - 本技能不依賴任何外部特定的 Git 工具或版控 Skill。在自身工作職責完成後即合規結束本任務。

## 預設載入: 環境規範

1. **[環境自適應]**: 必須先偵測環境 (PS1/SH)，選擇對應的沙盒腳本。
2. **[禁止猜測原則]**: 絕對嚴禁預估目標技能的行為，必須以讀取 `SKILL.md` 為唯一信源。

## 第一條：安全與執行體驗守則 (Safety & UX Protocols)

1. **[確認優先]**: 執行測試前，必須列出本次測試案例清單，讓使用者確認後才執行。
2. **[指令預告]**: 每一階段開始前，必須告知使用者「接下來要做什麼」。
3. **[逾時與重試]**: 交互逾時 10 秒應自主重試。沙盒執行最長容許 60 秒 Timeout。長時腳本必須保持 **Progress Echo** 輸出。任務自癒上限 **3 次**，超限觸發 **[RULE-005]** 停機守則。

## 第二條：測試邏輯規範

1. **[測試案例來源]**: 測試案例必須源自目標 Skill 的 `SKILL.md` 守則條文，逐條對應生成。
2. **[測試案例格式]**: 每一個測試案例必須包含：`ID`、`描述`、`預期行為`、`實際結果`、`是否通過`。
3. **[報告輸出]**: 完成後以結構化表格輸出測試報告，並記錄到 `測試記錄.md` 或工程師指定位置。

## 第三條：跨平台資源

- 核心工具：`bash`, `PowerShell`
- 腳本：`check-env.sh / .ps1`（環境檢測），`sandbox-init.sh`（沙盒建立），`sandbox-cleanup.sh`（清理）
