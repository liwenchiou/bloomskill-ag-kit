# 階段二：測試案例生成 (/develop)

## 執行邏輯

1. **環境前置檢查**:
   - 執行 `scripts/check-env.sh` 或 `.ps1`，確認沙盒所需環境齊全。
   - 執行 `scripts/sandbox-init.sh`，於 `/tmp/skill-sandbox/` 建立測試隔離目錄。
2. **生成測試案例**:
   - 依照 `/ideate` 確認的每一條 AC，生成對應的測試案例，格式如下：

   ```markdown
   | ID     | 描述                         | 測試方式                           | 預期行為             |
   |--------|------------------------------|------------------------------------|----------------------|
   | TC-001 | [確認優先] 守則驗證          | 模擬發出未確認指令                 | Agent 應先詢問確認   |
   | TC-002 | [逾時重試] 守則驗證          | 模擬 10 秒無回應                   | Agent 應自動重試     |
   | TC-003 | [RULE-006] 原子化限制驗證    | 模擬超過 5 個檔案的修改請求        | Agent 應建議拆分任務 |
   ```

3. **⛔ [強制停頓 - Phase Gate]**:
   - 列出所有生成的測試案例，詢問：**「測試案例已生成，是否同意我進行合規審查 (/validate)？」**
