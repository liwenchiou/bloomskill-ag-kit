# 階段一：需求定義與 Skill 解析 (/ideate)

## 執行邏輯

1. **詢問目標**:
   - 向使用者詢問：**「您想測試哪一個 Skill？請輸入技能名稱（對應 `.agent/skills/` 下的資料夾名稱）。」**
2. **讀取憲章**:
   - 讀取 `.agent/skills/<目標技能>/SKILL.md`，解析所有守則條文。
   - 同時讀取 `.agent/skills/<目標技能>/workflows/` 下的所有工作流檔案，了解每個階段預期行為。
3. **生成接收標準 (AC)**:
   - 依照憲章條文，列出本次測試的 **Acceptance Criteria 清單**，例如：
     - `AC-01`: 執行時有「確認優先」行為？
     - `AC-02`: 遇到超時是否重試？
     - `AC-03`: 是否每個階段都有「指令預告」？
4. **⛔ [強制停頓 - Phase Gate]**:
   - 展示 AC 清單，詢問使用者：**「以上測試標準是否符合您的期望？若無問題，請允許我進入 /develop 生成測試案例。」**
   - 在獲得明確同意前，**嚴禁進入下一階段**。
