# 階段四：沙盒執行與結果驗證 (/test)

## 執行邏輯

1. **沙盒執行**:
   - 在 `/tmp/skill-sandbox/` 中，逐一執行或模擬測試案例。
   - 每隔 3 秒輸出 **Progress Echo**：`⏳ 測試進行中 [TC-XXX] ...`
   - **每個 shell 指令 Timeout 上限為 10 秒**。若超過 10 秒無回應，必須立即中斷該指令，切換為替代方案（如改用 `grep_search` 工具取代 shell `grep`），並回報超時事件。
   - **⚠️ 嚴禁對含中文字元的關鍵字使用 shell `grep` 指令**。中文字元掃描一律改用 `grep_search` 工具，避免 locale 問題導致靜默卡死。
2. **自動錯誤排除**:
   - 若測試案例失敗或指令超時，自動解析原因，退回 `/develop` 修正，重試上限 **3 次**。
   - 超過重試限制時，立即輸出 `[ERROR_LOG]` 並觸發 **[RULE-005]** 停機。
3. **📋 [測試報告輸出]**:
   - 以結構化表格輸出最終結果：

   ```markdown
   | ID     | 描述                      | 預期行為           | 實際結果             | 通過？ |
   |--------|---------------------------|--------------------|----------------------|--------|
   | TC-001 | [確認優先] 守則驗證       | Agent 先詢問確認   | ✅ 符合預期          | ✅     |
   | TC-002 | [逾時重試] 守則驗證       | Agent 自動重試     | ✅ 符合預期          | ✅     |
   ```

   - 詢問使用者：**「要將此報告儲存到專案記錄檔嗎？（如：`測試記錄.md`）」**
4. **🧹 [環境清理]**:
   - 執行 `scripts/sandbox-cleanup.sh`，移除 `/tmp/skill-sandbox/` 所有臨時檔案。
   - 宣告：**「`skill-tester` 已完成本次測試任務，沙盒環境已清理完畢。」**
