# 階段二：代碼生成與腳本撰寫 (/develop)

在使用者確認藍圖後，依據 `/ideate` 發展出的設計，開始寫入具體的程式邏輯。

## 執行邏輯

1. **環境前置檢查**: 執行 `scripts/check-env.sh` 或 `.ps1` 確保所需工具齊全。
2. **高速實作**: 
   - 依照憲章「先斬後奏」的精神，直接產生、修改或刪除必要的檔案。
   - 保證代碼品質，適時加上註解，遵循既有的設計模式。
3. **⛔ [強制停頓 - Phase Gate]**: 
   - 向使用者列出剛才修改了哪些檔案與區塊。
   - 輸出：**「我已完成代碼撰寫，您想親自檢閱代碼，還是回覆『進行』以同意我進行安全性審查 (/validate)』
   - **在收到使用者明確的正向授權前，嚴禁呼叫任何下一階段的工具。**
