# 階段一：需求定義與架構設計 (/ideate)

此工作流負責在動手寫程式之前，先將使用者的模糊需求轉化為清晰的設計藍圖。

## 執行邏輯

1. **需求分析**: 
   - 梳理目前現有的目錄結構或專案脈絡。
   - 列出即將開發的目標與邊界，確保不涉及非必要修改。
2. **架構草案**: 
   - 擬定需要新增或修改的檔案清單。
   - 設計核心資料結構與依賴關係。
3. **⛔ [強制停頓 - Phase Gate]**:
   - 向使用者回報上述藍圖與可能的風險。
   - 輸出：**「上述設計是否符合您的預期？請回覆『進行』以允許我進入 /develop 階段，否則我將保持等待。」**
   - **在收到使用者明確的正向授權前，嚴禁呼叫任何下一階段的工具或進行任何檔案寫入。**
