# 階段四：沙盒執行與結果驗證 (/test)

確認過代碼與安全性後，進入最後的實地測試與拋轉確認環節。

## 執行邏輯

1. **實地測試**: 
   - 根據設計呼叫相關單元測試、CLI 指令，或以假資料 (Mock Data) 驗證功能。
2. **自動錯誤排除**: 
   - 若執行失敗，自動解析錯誤與追蹤 (Stack Trace)。
   - 以超時重試機制為限（最多不超過兩次），自動修正並回報給使用者。
   - 三段式防護：嘗試修復 → 若仍不成功 → 即刻中斷並提供排障建議給使用者。
3. **🎉 [任務完成與交接]**: 
   - 測試順利通過後，依循〈獨立封裝與獨立運作〉之約定宣告完工。
   - 收尾：「**我已經完成測試，功能運作正常！請確認所有的產出內容。若無問題，請允許我交出控制權，或者您可以呼叫 /git-manager 開始進行版控提交流程。**」
