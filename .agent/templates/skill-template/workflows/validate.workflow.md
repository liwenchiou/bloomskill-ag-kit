# 階段三：安全性與規範審查 (/validate)

在此階段，Agent 放下工程師帽子，戴上審查員的帽子，對剛剛寫成的系統進行靜態檢查。

## 執行邏輯

1. **靜態分析**: 
   - 審查各檔案是否符合《{{SKILL_NAME}} 核心憲章》以及整體專案規範。
   - 確認是否有潛在的安全性漏洞或越權行為（如擅自執行不可控的腳本）。
2. **依賴關係檢查**: 
   - 確保所有外部依賴皆已正確引導或文件化。
3. **⛔ [強制停頓 - Phase Gate]**: 
   - 將潛在的警告及防呆警示列出，讓使用者一目了然。
   - 如果代碼品質達標，輸出：**「已通過審查。請回覆『進行』以允許我將這些代碼放入沙盒進行實際執行測試 (/test)，否則我將保持等待。」**
   - **在收到使用者明確的正向授權前，嚴禁呼叫任何下一階段的工具。**
